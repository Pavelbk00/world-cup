# 📉 ПРАВИЛА ЭКОНОМИИ ТОКЕНОВ И ИСКЛЮЧЕНИЯ БОЛТОВНИ (.clinerules)

## 1. Режим ответа (Strict No Chitchat)

- **Прямой ответ сразу:** Запрещены приветствия, похвала, вежливость («Отличный выбор», «Рад помочь») и вводные фразы. Начинай ответ с кода или сути дела с первой строки.
- **Минимум текста:** Ограничивай пояснения. Длина предложений — строго до 10 слов. Без теории футбола и разжевывания базового синтаксиса JS/CSS.
- **Без вопросов в конце:** Запрещено предлагать улучшения, варианты развития и задавать вопросы («Что делаем дальше?», «Хотите перепишем?»). Выдал код — закрыл ответ.

## 2. Экономия токенов в коде

- **Только Diff:** Никогда не переписывай весь файл целиком ради исправления нескольких строк. Выдавай строго измененный фрагмент кода.
- **Один вариант:** Не генерируй альтернативы. Выдавай ровно один оптимальный вариант решения.
- **Без комментариев:** Запрещено писать комментарии внутри кода (`// ...`), если названия переменных отражают суть.
- **Сжатые примеры:** В примерах JSON/массивов показывай строго один элемент (один матч, один игрок). Не дублируй данные для всех 7 участников.

## 3. Архитектурные ограничения проекта

- **Только плоский код:** Запрещены вложенные таблицы (`<table>` в `<table>`). Используй CSS Grid, Flexbox и одномерные массивы.
- **Компактная логика:** Для расчета очков используй тернарные операторы и стрелочные однострочники вместо развесистых `if/else`.
- **Минимализм стека:** Стек зафиксирован (Vite, TS, Node.js, Express, локальный JSON). Любые предложения по усложнению (БД, Docker, Cron, сторонние тяжелые либы) отклоняются автоматически.

## 4. Контекст и Память проекта (Context Locking)

- **Запрет на повторный сбор контекста:** Запрещено выполнять команды `ls`, `find`, парсить заново структуру папок или читать файлы конфигурации, если это уже делалось в рамках текущей сессии.
- **Жёсткий слепок архитектуры:** Запомни и никогда не переспрашивай:
  - Количество игроков: строго 7 человек (Должик, Лёха, Пашок, Роман, Шевыч, Хлам, Лебедь).
  - Хранилище: только локальные JSON-файлы в папке `data/` (базы данных нет).
  - Стек: бэкенд на Node.js (Express), фронтенд на Vite (TypeScript), сервер Ubuntu.
- **Использование кэша:** При изменении логики или верстки опирайся на уже известные тебе структуры файлов `predictions.json` и `groups.json`. Не запрашивай их содержимое повторно, если структура не менялась.

## 5. Правила турнира и начисления очков (Tournament Rules)

- **1. Прогнозы на матчи:** Основное время (90 минут + добавленное). Исход (П1, Х, П2) = 1 очко; угаданная разница мячей = +2 очка; точный счет = +3 очка. Максимум за матч = 6 очков.
- **2. Итоги группового этапа:** За каждую команду, вышедшую в 1/16 = 3 очка; за точное место (1, 2, 3, 4) = 2 очка. Максимум за одну команду = 5 очков.
- **3. Плей-офф (способ победы):** Бонус за способ прохода (только если угадан победитель): в основное время = +1 очко; в дополнительное время = +3 очка; в серии пенальти = +5 очков.
- **4. Лучший бомбардир:** Ставка до старта турнира. За "Золотую бутсу" ФИФА = 20 очков; за каждый гол игрока (без серий пенальти) = 2 очка.
- **5. Призеры турнира:** Ставка на топ-3 до старта турнира. За угаданное Золото = 50 очков; Серебро = 35 очков; Бронза = 20 очков.
